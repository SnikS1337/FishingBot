# FishingBot — дизайн-спецификация (MVP v1)

## 1) Цель

Собрать десктоп-приложение на **C# .NET 8 + WPF** для автоматизации рыбалки в игровом интерфейсе с полным циклом:

`детект подсказки "Нажмите E" → старт рыбалки (E) → ожидание клева → подсечка → борьба с рыбой (A/D) → меню поимки (забрать/отпустить) → перезаброс`.

Дополнительно: полноценный **GUI с live-debug** и безопасный режим калибровки **DetectOnly** (без эмуляции ввода).

---

## 2) Зафиксированные решения

- Подход реализации: **C (сбалансированный модульный MVP)**
- GUI: **WPF**
- Режимы: **DetectOnly** и **Active**
- Приоритет качества: **баланс надежности и скорости**
- Захват экрана: **DXGI Desktop Duplication / Windows Graphics Capture подход**
- Ввод: **SendInput + hardware scan-коды**
- Мышь: **человекоподобная траектория (Bezier)**

---

## 3) Архитектура

## 3.1 Компоненты

1. **CaptureEngine**
   - Получение кадров экрана через DXGI.
   - Отдача кадров в pipeline в формате, удобном для OpenCV.

2. **VisionPipeline**
   - Детекторы игровых сигналов:
     - `StartPromptDetector` (подсказка в левом верхнем углу: «Нажмите E чтобы начать рыбалку»)
     - `TensionDetector` (виджет «Натяжение»)
     - `FightDetector` (нижняя шкала борьбы + трекинг маркера/рыбы)
     - `CatchMenuDetector` (центральное окно поимки и кнопки)
   - Каждый детектор возвращает `isDetected`, `confidence`, метаданные.

3. **FishingStateMachine**
   - Управляет состояниями и переходами по событиям детекторов + таймерам.

4. **InputEngine**
   - Клавиши: E / Space / A / D через scan-коды.
   - Удержание/отпуск/переключение A/D.
   - Клик по кнопкам меню через Bezier-мышь + случайные задержки.

5. **AppOrchestrator**
   - Связывает GUI, FSM, vision и input.
   - Управляет запуском/остановкой/паузой/panic.

6. **ConfigService**
   - Загрузка/сохранение профиля `config.json`.
   - Нормализованные ROI и тайминги.

7. **WPF GUI (MVVM)**
   - Управление режимами.
   - Live-превью и debug-оверлеи.
   - Калибровка ROI и журнал событий.

## 3.2 Поток данных

`CaptureEngine -> VisionPipeline -> FSM -> (в Active) InputEngine -> FSM`

GUI подписывается на телеметрию (state, confidence, fps, лог действий).

## 4) Машина состояний (MVP)

```text
STOPPED
  -> IDLE
      -> WAIT_START_PROMPT
          -> START_FISHING (E)
              -> WAIT_BITE
                  -> BITE_READY
                      -> HOOK (Space)
                          -> FIGHT
                              -> CATCH_MENU
                                  -> APPLY_ACTION (TAKE/RELEASE)
                                      -> RECAST
                                          -> WAIT_START_PROMPT
```

### Семантика состояний

- **WAIT_START_PROMPT**: ожидаем и подтверждаем подсказку «Нажмите E чтобы начать рыбалку».
- **START_FISHING**: однократное нажатие `E` для входа в рыбалку.
- **WAIT_BITE**: виджет «Натяжение» наблюдается в нейтральном состоянии.
- **BITE_READY**: подтверждено состояние «клюет» (красный индикатор/сигнал).
- **HOOK**: однократная подсечка (`Space`) с короткой рандомной задержкой.
- **FIGHT**: управление A/D по направлению движения маркера.
- **CATCH_MENU**: обнаружено окно поимки и кнопки действия.
- **APPLY_ACTION**: клик по `TAKE` или `RELEASE`.
- **RECAST**: пауза и следующий заброс.

## 5) Детект-логика

## 5.1 ROI-модель

Все регионы хранятся в **нормализованных координатах** (`0..1`) и конвертируются в пиксели под текущее разрешение.

Причина: переносимость между 1920x1080, 2560x1440 и разными UI-scale.

## 5.2 Детекторы

1. **TensionDetector**
   - Анализ ROI виджета `Натяжение` справа снизу.
   - Признак `bite`: преобладание красного в контрольной полосе + стабильность N кадров.

2. **StartPromptDetector**
   - Анализ ROI левого верхнего угла для подсказки `Нажмите E чтобы начать рыбалку`.
   - Детект по комбинации визуальных признаков (текст/плашка) с подтверждением N кадров.

3. **FightDetector**
   - Детект появления fight-шкалы в нижней части.
   - Трекинг X-позиции маркера/рыбы.
   - Расчет `deltaX` относительно предыдущего кадра.

4. **CatchMenuDetector**
   - Детект центрального окна поимки (контрастный прямоугольный блок).
   - Уточнение позиции кнопок `Забрать себе` / `Отпустить` через шаблоны.

## 5.3 Устойчивость детектов

- Переходы FSM по событию только после **N последовательных подтверждений**.
- Используется порог `confidence >= threshold`.
- На случай дрожания UI: сглаживание кратковременных выпадений (debounce).

---

## 6) Логика ввода

## 6.1 Клавиатура

- `E`: одиночное нажатие (down/up) на этапе старта рыбалки
- `Space`: одиночное нажатие (down/up)
- `A`/`D`: удержание и переключение по направлению движения маркера
- Гарантированный отпуск всех клавиш при `Stop`, `Panic`, аварийных ветках

## 6.2 Мышь

- Перемещение по кривой Безье с рандомизацией контрольных точек и скорости
- Небольшая случайная задержка перед кликом в меню поимки

## 6.3 Режимы безопасности

- **DetectOnly**: никакой эмуляции ввода
- **Active**: полный ввод

## 7) GUI (WPF)

## 7.1 Основные панели

1. **Control**
   - `Start DetectOnly`, `Start Active`, `Pause`, `Stop`
   - Выбор действия после поимки: `TAKE/RELEASE`

2. **Live View**
   - Превью текущего кадра (downscale)
   - Наложение ROI и отладочных меток
   - Отображение confidence и FPS
   - Индикатор `StartPrompt` (detected/not detected)

3. **Calibration**
   - Ручная правка ROI (перемещение/изменение размера)
   - Применение и сохранение профиля
   - Тест детекторов на текущем кадре

4. **Logs**
   - Лента событий FSM и действий ввода
   - Сообщения ошибок и предупреждений

## 7.2 Hotkeys

- `F9` — Pause/Resume
- `F10` — Panic Stop

---

## 8) Конфигурация

`config.json` содержит:

- Параметры разрешения/режима
- Нормализованные ROI
- Тайминги (интервалы детекта, диапазоны задержек)
- Пороги confidence и N подтверждений
- Отдельные пороги/параметры для `StartPromptDetector`
- Выбор `TAKE/RELEASE`
- Включение/выключение debug-режимов

---

## 9) Ошибки и восстановление

- Таймаут каждого ключевого состояния (`WAIT_START_PROMPT`, `WAIT_BITE`, `FIGHT`, `CATCH_MENU`)
- При таймауте: безопасный reset в `WAIT_START_PROMPT` или `IDLE`
- Всегда выполнять `KeyUp(E/A/D/Space)` перед reset
- При невалидном кадре захвата: мягкий ретрай без краша UI

---

## 10) Тестовая стратегия (MVP)

1. **Unit-тесты**
   - Переходы FSM
   - Таймауты и защитные ветки
   - Конвертация normalized ROI -> pixel ROI

2. **Интеграционные офлайн-тесты**
   - Детекторы на наборе эталонных скриншотов
   - Проверка confidence и стабильности N-подтверждений

3. **Runtime-валидация**
   - Проверка hotkeys/panic
   - Отсутствие залипания клавиш
   - Стабильность цикла в DetectOnly и Active

---

## 11) Критерии приемки MVP

MVP считается готовым, если выполняются условия:

1. Приложение запускается на .NET 8 (Windows) и открывает WPF GUI.
2. В DetectOnly режиме корректно видны ROI, confidence, live логика переходов.
3. В Active режиме проходит полный цикл рыбалки (`StartPrompt -> E -> подсечка -> fight -> меню -> действие -> перезаброс`).
4. Hotkeys `F9/F10` работают стабильно в любой фазе.
5. После stop/panic нет залипших клавиш.
6. Конфиг сохраняется/загружается без потери параметров.

---

## 12) Что не входит в MVP (чтобы не раздувать scope)

- Поддержка нескольких игр/разных UI-паков “из коробки”
- Обучаемые ML-модели детекта
- Распределенная/сетевые сценарии
- Сложный multi-profile orchestration
